## Repository Task

Search the repository for domain terms from the ticket. Inspect related code,
configuration, tests, and documentation before summarizing context. Separate
confirmed context from assumptions.

## Report

Produce one concise Markdown context report with:

- Ticket summary.
- Relevant files and why each file matters.
- Existing behavior.
- Tests or test gaps.
- Questions or missing information.
- Likely entry points, if useful.

Do not include:

- Code edits.
- An implementation plan.
- Generic project commentary.
- Unverified assumptions presented as facts.
- Broad rewrites unrelated to the ticket.
