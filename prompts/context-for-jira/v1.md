## Task

Collect repository context related to this ticket.

Focus on:

- What the ticket appears to ask for.
- Relevant files and why each file matters.
- Existing behavior inferred from code, configuration, and documentation.
- Tests or test gaps related to the ticket.
- Questions or missing information.
- Likely implementation entry points when appropriate.

## Output Constraints

Produce one concise Markdown context report.

The report should include:

- Ticket summary.
- Relevant files and why each file matters.
- Existing behavior.
- Tests or test gaps.
- Questions or missing information.
- Likely entry points, if useful.

The report must not include:

- Code edits.
- An implementation plan.
- Generic project commentary.
- Unverified assumptions presented as facts.
- Broad rewrites unrelated to the ticket.
